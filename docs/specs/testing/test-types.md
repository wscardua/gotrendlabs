# Tipos de Teste

## Unitários

- validação de stake
- regras de status de mercado
- lançamentos de ledger
- cálculo de elegibilidade para ações administrativas

## Integração

- criação de previsão e persistência associada
- resolução de mercado e efeitos em wallet/reputação
- integração entre backend e comunicações
- consultas do frontend aos contratos principais

## Fluxo / End-to-End

- cadastro até primeira previsão
- mercado aberto até resolução
- visão de wallet e extrato
- envio de sugestão e tratamento administrativo

## Regressão

- cada contrato alterado deve listar cenários críticos protegidos
- estados antigos não devem sumir sem migração explícita
